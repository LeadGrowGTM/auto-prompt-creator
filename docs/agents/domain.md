# Domain Layout

## Context model: Multi-context

This repo is **multi-context** — it hosts multiple independent optimization scenarios, each with its own rubric, data splits, prompt versions, and eval history. Contexts do not share ground truth or prompts.

```
scenarios/
  icp-classification/     <- Context A: B2B lead qualification
  prospect-identification/ <- Context B: Buyer identity extraction
  b2b-robotics-qualifier/ <- Context C: Robotics vertical qualifier
  rendered-semantic-qa/   <- Context D: Semantic QA for rendered output
  skill-invoker/          <- Context E: Claude Code skill routing
```

Each scenario context is self-contained:
- Its own `scenario.json` (rubric, thresholds, model)
- Its own `ground-truth/` (expert-validated outputs)
- Its own `evals/loop-state.json` (iteration history, halt state)
- Its own `prompts/v001.md ... vNNN.md` (prompt lineage)

## Shared infrastructure (repo-level)

| File | Role |
|------|------|
| `setup.mjs` | Scaffolds new scenario contexts from CLI args |
| `setup.test.mjs` | Tests the scaffolder — the only test suite in the repo |
| `library/` | Cross-context — graduated prompts land here with metadata |
| `METHODOLOGY.md` | Cross-context — the optimization rules that all scenarios follow |
| `.claude/skills/anneal-prompt/SKILL.md` | Cross-context — Claude Code skill that drives the loop |

## Where domain docs should live

| Doc type | Location |
|----------|----------|
| Scenario-specific notes | `scenarios/[name]/README.md` (auto-generated by setup) |
| Scenario config | `scenarios/[name]/scenario.json` |
| Cross-scenario methodology | `METHODOLOGY.md` |
| Graduated prompt metadata | `library/[scenario].md` YAML frontmatter |
| Engineering architecture | `docs/architecture/` (this docs tree) |
| Domain glossary | `docs/domain/glossary.md` |
| Architecture decisions | `docs/domain/adr/` |
